Security architecture

Private support context with explicit operational controls.

SupportMemory is designed to keep organisational context scoped, external events authenticated, connector credentials protected, and actions accountable.

Identity

Authenticated context

Private APIs resolve organisation, workspace, role, and scopes from an active access key rather than trusting browser-provided tenancy.

Secrets

Encrypted credentials

Integration credentials are encrypted before database persistence and are never returned through administration responses.

Ingress

Verified webhooks

Provider signatures, timestamps, replay windows, and deterministic event identifiers protect inbound connector traffic.

Access

Role-based permissions

Owner, administrator, operator, auditor, and viewer roles receive distinct read, management, and delivery capabilities.

Delivery

Controlled egress

HTTPS enforcement, REST host allowlists, idempotency, rate-limit handling, and retry policies constrain outbound work.

Audit

Operational records

Connector setup, tests, inbound processing, outbound delivery, failures, and administrative changes create audit events.

Deployment responsibility

Production controls remain environment-specific.

Production deployments must provide strong signing and encryption keys, enable authentication, restrict approved REST hosts, configure provider OAuth applications, terminate TLS at the deployment edge, and operate retention and incident-response processes appropriate to the organisation.
