System Capabilities

Three core breakthroughs that separate SupportMemory from standard LLM wrapper pipelines.

Core Module 01

Persistent Memory

Every interaction is vectorized. New tickets trigger semantic recall across that customer and similar issues elsewhere — before the agent replies.

  • Customer preferences & SLA rules
  • Past solutions & resolution notes
  • Ticket-to-ticket vector correlation
  • Account configuration state
  • Internal engineering notes
POST /api/kb/search
const hits = await kb.search({
  query: "webhook 401 after secret rotation",
  customer_id: "cust_apex",
  semantic_threshold: 0.84,
  limit: 8
});

// Context Health filters stale chunks
return { loaded: hits.length, top: hits[0] };

Core Module 02

Checkpoint Recovery

Every investigation step checkpoints to PostgreSQL. Interruptions resume without duplicate refunds, emails, or tool charges.

  • API call payloads & results preserved
  • Reasoning step history retained
  • Duplicate outbound actions prevented
  • Signed execution receipt generated
Core Module 03

Continuous Learning

After each ticket: reflect → curate → retrieve. Lessons become playbook rules for the next similar case.

  • 1. Reflect on the execution path
  • 2. Curate versioned lessons
  • 3. Retrieve before the next run

Task contract

Durable scope for each investigation — what the agent may do and what counts as done.

Runtime Governor

Hybrid PII: redact on reads; require approval on external send/refund tools.

Execution receipt

SHA-256 hashed, Ed25519-signed proof of tools, context, checkpoint, and recovery.
