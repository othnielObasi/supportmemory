Questions before deployment

What organisations need to know.

Does SupportMemory replace Zendesk, Intercom, or Freshdesk?+

No. Those systems continue to manage support channels and agent workflows. SupportMemory adds durable cross-case context, governed organisational knowledge, recoverable investigations, and linked execution evidence.

What does SupportMemory store?+

Depending on deployment configuration, it stores tenant-scoped conversations, verified knowledge sources, graph entities and relationships, checkpoints, memory records, connector events, delivery state, and audit evidence.

How does knowledge-graph memory help?+

It connects customers, symptoms, products, policies, cases, and resolutions so an investigation can follow relevant relationships and preserve the evidence behind the path—not only retrieve similar text.

Are the listed integrations live or placeholders?+

The application includes provider configuration, webhook handling, connection testing, delivery state, and administration surfaces for Zendesk, Intercom, Freshdesk, Slack, and REST. Each deployment still requires real provider credentials and configuration.

Can an organisation self-host it?+

The repository includes containerised application services. Production operators remain responsible for infrastructure hardening, secrets, TLS, data policy, monitoring, backups, provider agreements, and operational support.
